# Messaging Framework (제품 메시징 및 포지셔닝)

## 트리거
- 제품의 핵심 메시지를 정의해야 할 때
- "메시징", "포지셔닝", "messaging", "positioning", "가치 제안" 키워드가 언급될 때
- 마케팅 채널별 메시지 변형이 필요할 때
- 경쟁사 대비 차별화 메시지를 수립해야 할 때

## 동작
1. **포지셔닝 전략 수립**:
   - April Dunford의 Obviously Awesome 프레임워크 적용
   - 경쟁 대안 정의
   - 고유한 속성 (Unique Attributes) 식별
   - 속성이 제공하는 가치 연결
   - 최적 타겟 고객 세그먼트 확인
   - 시장 카테고리 선정
2. **메시지 계층 구조 설계**:
   - 브랜드 포지셔닝 스테이트먼트
   - 엘리베이터 피치 (30초/60초)
   - 핵심 가치 제안 (1문장)
   - 3-4개 지원 메시지 (Supporting Messages)
   - 각 지원 메시지의 증거 포인트 (Proof Points)
3. **페르소나별 메시지 맞춤화**:
   - ICP 내 각 페르소나의 관심사에 맞는 메시지 변형
   - 의사결정자용 vs 사용자용 메시지 구분
   - 기술적 메시지 vs 비즈니스 메시지 구분
4. **채널별 메시지 최적화**:
   - 웹사이트 히어로 섹션
   - 앱스토어 설명
   - 소셜 미디어 (LinkedIn, Twitter)
   - 이메일 시퀀스
   - 영업 자료 (데크, 원페이저)
5. **메시지 테스트 계획**:
   - A/B 테스트 가능한 메시지 변형 제안
   - 메시지 효과 측정 메트릭
   - 반복 개선 프로세스

## 출력
- 포지셔닝 캔버스
- 메시지 계층 구조 문서
- 페르소나별 메시지 매트릭스
- 채널별 메시지 가이드
- 엘리베이터 피치 및 태그라인 후보

## 참고
- Dart 풀스택 (Flutter + Jaspr + Serverpod)의 기술적 차별점을 비기술자에게 설명하는 메시지 필요
- 크로스 플랫폼 (모바일 + 웹) 동시 제공이라는 고유 가치를 메시지에 반영
- 앱스토어 (iOS/Android)와 Jaspr Web 랜딩 페이지의 메시지 톤 차이 고려
- BMAD의 기능 명세와 별개로, 외부 커뮤니케이션 관점의 메시지에 집중
