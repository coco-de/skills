# GTM Motion (Go-to-Market 모션 계획)

## 트리거
- 제품 출시 또는 시장 진입 전략을 수립해야 할 때
- "GTM", "go-to-market", "런칭", "출시 전략" 키워드가 언급될 때
- 새로운 시장/세그먼트로 확장을 계획할 때
- 채널 전략 또는 성장 전략이 필요할 때

## 동작
1. **GTM 모션 유형 선택**:
   - PLG (Product-Led Growth): 제품 자체가 성장 엔진
   - SLG (Sales-Led Growth): 영업 팀 중심 성장
   - CLG (Community-Led Growth): 커뮤니티 기반 성장
   - 하이브리드 모델 설계
   - 각 모션의 장단점 및 적합성 분석
2. **런칭 전략 수립**:
   - 런칭 티어 결정 (Soft Launch / Beta / GA / Big Bang)
   - 런칭 타임라인 및 마일스톤
   - Pre-launch 활동 (대기자 명단, 얼리 액세스, 베타 프로그램)
   - Launch Day 체크리스트
   - Post-launch 모니터링 계획
3. **채널 전략**:
   - 획득 채널 매핑 (유료/무료, 인바운드/아웃바운드)
   - 채널별 CAC 예측 및 비교
   - 채널 우선순위화 (ICE 스코어링)
   - 채널별 콘텐츠 전략
4. **GTM 퍼널 설계**:
   - 인지 (Awareness) → 관심 (Interest) → 고려 (Consideration) → 전환 (Conversion) → 활성화 (Activation)
   - 각 단계별 핵심 메트릭
   - 전환율 목표 설정
   - 병목 구간 예측 및 대응 전략
5. **성장 실험 프레임워크**:
   - 그로스 루프 (Growth Loop) 식별
   - 바이럴 계수 목표 설정
   - 레퍼럴 프로그램 설계
   - 초기 성장 레버 우선순위화

## 출력
- GTM 전략 문서 (모션 유형, 채널, 타임라인)
- 런칭 플레이북 (체크리스트 포함)
- 채널 전략 매트릭스
- GTM 퍼널 및 KPI 대시보드 설계
- 성장 실험 백로그

## 참고
- Flutter 앱은 앱스토어 최적화 (ASO)가 핵심 채널, Jaspr Web은 SEO가 핵심 채널
- 앱스토어 리뷰 관리와 웹 사이트 전환율 최적화를 동시에 고려
- Serverpod 기반의 서버 확장 계획이 GTM 스케일업과 연동되어야 함
- PLG 모션 선택 시 제품 내 온보딩, 셀프서브 기능이 Dart 풀스택으로 구현 가능
