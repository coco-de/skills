# Business Model (비즈니스 모델 분석)

## 트리거
- 제품의 수익 모델을 설계하거나 검토해야 할 때
- "비즈니스 모델", "수익 모델", "유닛 이코노믹스", "unit economics" 키워드가 언급될 때
- 가격 정책 또는 수익화 전략을 수립해야 할 때
- 투자 유치 또는 사업 계획서 작성 시

## 동작
1. **비즈니스 모델 캔버스 작성**:
   - 고객 세그먼트, 가치 제안, 채널, 고객 관계
   - 수익원, 핵심 자원, 핵심 활동, 핵심 파트너
   - 비용 구조
2. **수익 모델 설계**:
   - 적합한 수익 모델 유형 분석 (SaaS, 프리미엄, 거래 수수료, 광고 등)
   - 가격 티어 구조 설계
   - 무료/유료 경계선 정의 (프리미엄 모델의 경우)
3. **유닛 이코노믹스 분석**:
   - CAC (고객 획득 비용) 산출
   - LTV (고객 생애 가치) 산출
   - LTV/CAC 비율 분석
   - Payback Period 계산
   - 마진 구조 분석
4. **재무 모델링**:
   - 매출 예측 시나리오 (베이스/낙관/비관)
   - 손익분기점 분석
   - 코호트 기반 매출 모델링
5. **모델 검증 포인트 도출**:
   - 핵심 가정 식별 (WTP, 전환율, 이탈률)
   - 검증 우선순위 및 방법 제안

## 출력
- 비즈니스 모델 캔버스 (Markdown 테이블)
- 수익 모델 설계서
- 유닛 이코노믹스 분석표
- 재무 예측 시나리오
- 가격 정책 제안서

## 참고
- Serverpod 호스팅 비용이 서버 비용 구조에 영향 (자체 호스팅 vs 클라우드)
- Flutter 앱의 앱스토어 수수료 (30%) 반영 필요
- Jaspr Web은 추가 배포 비용 없이 웹 접근성 확보 가능
- 단일 Dart 코드베이스로 인한 개발 비용 절감 효과를 비용 구조에 반영
