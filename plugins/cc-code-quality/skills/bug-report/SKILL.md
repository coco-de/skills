# Bug Report

이미지 분석을 통한 버그 리포트 생성 및 ZenHub 이슈 자동화 스킬입니다.

## Scope and Capabilities

### 핵심 기능

| 기능 | 설명 |
|------|------|
| 이미지 분석 | 스크린샷에서 UI 상태, 에러 메시지 자동 추출 |
| 버그 분류 | 심각도 및 버그 유형 자동 분류 |
| 이슈 생성 | ZenHub/GitHub 이슈 자동 생성 |
| 라벨링 | 버그 특성에 따른 자동 라벨 지정 |

### 지원 버그 유형

| 유형 | 아이콘 | 설명 |
|------|--------|------|
| UI/UX | 🎨 | 레이아웃 깨짐, 스타일 오류 |
| 기능 | ⚙️ | 버튼 미동작, 로직 오류 |
| 성능 | ⚡ | 느린 로딩, 프레임 드롭 |
| 크래시 | 💥 | ANR, 앱 강제 종료 |
| 데이터 | 📊 | 잘못된 데이터, 동기화 오류 |

## Quick Start

### 이미지와 함께 버그 리포트

```bash
/bug-report /path/to/screenshot.png "로그인 버튼이 동작하지 않습니다"
```

### 텍스트만으로 버그 리포트

```bash
/bug-report "결제 완료 후 주문 내역이 표시되지 않음"
```

### 심각도 지정

```bash
/bug-report --severity critical "앱이 시작 시 크래시됩니다"
```

## 버그 리포트 프로세스

### 1단계: 이미지 분석

스크린샷이 제공된 경우:
- UI 상태 파악 (에러 다이얼로그, 빈 화면, 로딩 상태)
- 에러 메시지 텍스트 추출
- 버그 유형 자동 추론
- 영향받는 컴포넌트 식별

### 2단계: 정보 수집

```markdown
## 수집 정보

1. **재현 단계**: 버그를 재현하는 단계
2. **예상 결과**: 정상 동작 시 기대되는 결과
3. **실제 결과**: 현재 발생하는 문제
4. **환경 정보**: OS, 디바이스, 앱 버전
5. **추가 정보**: 로그, 스택트레이스 등
```

### 3단계: 이슈 생성

```markdown
## ZenHub 이슈 생성

- **제목**: [Bug] {버그 요약}
- **본문**: 구조화된 버그 리포트
- **라벨**: bug, severity:{level}, area:{affected_area}
- **파이프라인**: Triage
```

### 워크플로우 연계

버그 수정까지 자동화하려면:

```bash
# 버그 리포트 생성 후 바로 수정 사이클 시작
/workflow:bug-cycle --new /path/to/screenshot.png "버그 설명"

# 또는 기존 이슈로 수정 사이클 시작
/workflow:bug-cycle 123
```
