# Security Audit Checklist (보안 감사 체크리스트)

## 트리거
- 릴리스 전 보안 점검을 수행할 때
- 정기 보안 감사를 실시할 때
- 새로운 기능 추가 후 보안 영향을 평가할 때
- 의존성 패키지의 취약점을 스캔할 때

## 동작
1. OWASP Mobile Top 10 기반 점검을 수행한다
   - M1: 부적절한 자격 증명 사용
   - M2: 부적절한 공급망 보안
   - M3: 안전하지 않은 인증/인가
   - M4: 부적절한 입력/출력 검증
   - M5~M10: 안전하지 않은 통신, 개인정보 보호 등
2. OWASP Web Top 10 기반 점검을 수행한다 (Jaspr Web)
   - 인젝션, 인증 실패, 민감 데이터 노출 등
   - XSS, CSRF 방지 확인
   - 보안 헤더(CSP, HSTS 등) 설정 검증
3. 의존성 취약점을 스캔한다
   - `dart pub audit` 실행
   - 알려진 CVE 목록과 대조
   - 오래된 패키지 업데이트 권고
4. Serverpod 보안 설정을 검증한다
   - 엔드포인트 인증 누락 점검
   - 데이터베이스 접근 권한 확인
   - 로그 내 민감 정보 노출 점검

## 출력
- 보안 감사 체크리스트 결과 (항목별 통과/미통과/해당 없음)
- 취약점 심각도별 분류 (Critical, High, Medium, Low)
- 개선 권고 사항 및 우선순위
- 의존성 취약점 스캔 보고서

## 참고
- `dart pub audit`는 pub.dev의 보안 권고를 기반으로 의존성을 검사한다
- Cocode 프로젝트는 Serverpod, Flutter, Jaspr 모두의 의존성을 주기적으로 스캔해야 한다
- Clean Architecture 덕분에 보안 관련 로직이 특정 레이어에 집중되어 감사 범위를 명확히 할 수 있다
- 보안 감사 결과는 팀 내 공유하고, Critical/High 항목은 즉시 수정 일정을 잡는다
- CI/CD 파이프라인에 `dart pub audit`를 포함하여 매 빌드 시 자동 스캔한다
