# SQL Analysis (SQL 기반 제품 분석)

## 트리거
- 제품 데이터를 SQL로 분석해야 할 때
- "SQL 분석", "쿼리", "대시보드", "데이터 추출" 키워드가 언급될 때
- 제품 메트릭을 데이터베이스에서 직접 조회해야 할 때
- ClickHouse 기반 분석 쿼리가 필요할 때

## 동작
1. **분석 목표 정의**:
   - 비즈니스 질문을 SQL 질문으로 변환
   - 필요한 데이터 소스 및 테이블 식별
   - 기대 결과 형태 정의
2. **쿼리 설계 및 작성**:
   - ClickHouse 호환 SQL 문법 사용
   - 윈도우 함수 활용 (순위, 이동 평균, 누적 합계)
   - 날짜/시간 함수 (toDate, toStartOfWeek, dateDiff 등)
   - 집계 함수 및 그룹화 최적화
   - 서브쿼리 및 CTE (WITH절) 활용
3. **핵심 분석 패턴 제공**:
   - DAU/WAU/MAU 계산
   - 퍼널 분석 (단계별 전환율)
   - 리텐션 쿼리 (N-day, unbounded)
   - 매출 분석 (MRR, ARR, 이탈률)
   - 사용자 세그먼트별 행동 분석
4. **대시보드 설계**:
   - 핵심 메트릭 대시보드 구성 제안
   - 쿼리 → 차트 유형 매핑
   - 필터 및 드릴다운 구조 설계
5. **쿼리 최적화**:
   - ClickHouse 특화 최적화 (파티션 키 활용, 적절한 엔진 선택)
   - 대용량 데이터 처리 패턴
   - 물리화 뷰 (Materialized View) 활용 제안

## 출력
- SQL 쿼리 (ClickHouse 호환, 주석 포함)
- 쿼리 결과 해석 가이드
- 대시보드 설계 문서
- 데이터 딕셔너리 (관련 테이블/컬럼 설명)

## 참고
- Serverpod의 데이터 모델이 PostgreSQL 기반이므로, 분석용 ClickHouse로의 데이터 파이프라인 고려
- Serverpod ORM 모델과 분석 테이블 간의 매핑 관계 파악 필요
- Flutter/Jaspr 클라이언트에서 발생하는 이벤트 로깅 구조를 고려한 쿼리 설계
- Cocode 팀의 기존 ClickHouse 인프라와 호환되는 쿼리 작성
