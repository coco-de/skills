# Assumption Testing (가정 테스트)

## 트리거
- 새로운 제품/기능의 핵심 가정을 검증해야 할 때
- "가정 테스트", "assumption", "리스크 검증" 키워드가 언급될 때
- 제품 아이디어의 실행 가능성을 판단해야 할 때
- Leap of Faith 가정을 식별해야 할 때

## 동작
1. **가정 매핑**:
   - 제품/기능 관련 모든 가정을 나열
   - 가정 유형 분류: 가치 가정 (Desirability), 실행 가정 (Feasibility), 수익 가정 (Viability)
   - 각 가정의 근거 수준 평가 (추측/일화/데이터)
2. **위험도 평가**:
   - 영향도 x 불확실성 매트릭스로 우선순위화
   - "가장 위험한 가정" (Riskiest Assumption) 식별
   - 킬러 가정 (실패 시 전체 제품 무효화) 표시
3. **검증 방법 매칭**:
   - 각 가정에 적합한 검증 방법 추천
   - 데스크 리서치: 시장 데이터, 경쟁사 분석
   - 사용자 리서치: 인터뷰, 서베이
   - 기술 스파이크: PoC, 프로토타입
   - 시장 테스트: 랜딩 페이지, 사전 등록
4. **검증 계획 수립**:
   - 가정별 검증 실험 설계
   - 최소 비용/시간으로 검증 가능한 방법 우선
   - 검증 완료 기준 정의
5. **결과 기록 및 업데이트**:
   - 가정 상태 추적 (미검증/검증중/확인/반증)
   - 반증된 가정에 대한 피벗 옵션 제시

## 출력
- 가정 매핑 문서 (전체 가정 목록 + 유형 분류)
- 위험도 우선순위 매트릭스
- 가정별 검증 계획서
- 가정 추적 대시보드 템플릿
- 피벗/진행 의사결정 프레임워크

## 참고
- Dart 풀스택 환경에서 기술 실행 가정은 Flutter/Jaspr/Serverpod 기술 스택 제약을 반영
- 크로스 플랫폼 (모바일 + 웹) 동시 지원에 대한 기술 가정 검증 포함
- Serverpod의 ORM, 인증, 실시간 통신 기능 관련 기술 가정 고려
- BMAD의 요구사항 정의 전 단계로, 가정 검증 결과가 BMAD 입력으로 활용됨
