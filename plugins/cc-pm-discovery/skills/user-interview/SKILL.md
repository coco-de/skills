# User Interview (사용자 인터뷰)

## 트리거
- 새로운 기능이나 제품에 대한 사용자 니즈를 파악해야 할 때
- 기존 기능의 사용성 문제를 깊이 이해해야 할 때
- "사용자 인터뷰", "고객 인터뷰", "user interview" 키워드가 언급될 때
- 정성적 데이터 수집이 필요할 때

## 동작
1. **인터뷰 목표 설정**: 검증하려는 가설 또는 탐색하려는 영역을 명확히 정의
2. **질문 프레임워크 설계**:
   - The Mom Test 기법 적용 (구체적 과거 경험 중심 질문)
   - 개방형 질문 → 심화 질문 → 확인 질문 순서로 구성
   - 유도 질문 및 가설 확인 편향을 피하는 질문 작성
3. **인터뷰 가이드 작성**:
   - 도입부 (라포 형성, 컨텍스트 설정)
   - 핵심 질문 (5-7개)
   - 후속 질문 프롬프트
   - 마무리 (추가 의견, 다음 단계 안내)
4. **인터뷰 노트 템플릿 제공**: 관찰, 직접 인용, 감정 반응 기록 구조
5. **인사이트 종합**:
   - 패턴 및 테마 도출
   - 인사이트를 기회 영역으로 변환
   - 우선순위화 매트릭스 적용

## 출력
- 인터뷰 질문 가이드 (Markdown)
- 인터뷰 노트 템플릿
- 인사이트 종합 보고서 (패턴, 테마, 기회 영역)
- 후속 액션 아이템 목록

## 참고
- CoCode 팀은 Dart 풀스택 (Flutter + Jaspr Web + Serverpod)을 사용하므로, 기술적 제약 관련 질문도 포함 가능
- Flutter 앱과 Jaspr 웹 양쪽의 사용자 경험 차이를 탐색하는 질문 설계 고려
- Serverpod 백엔드 성능 관련 사용자 체감 질문 포함 가능
- 인터뷰 결과는 BMAD의 PRD가 아닌, 발견 단계의 원시 인사이트로 활용
