# ICP Definition (이상적 고객 프로필 정의)

## 트리거
- 타겟 고객을 명확히 정의해야 할 때
- "ICP", "이상적 고객", "타겟 고객", "고객 프로필" 키워드가 언급될 때
- 마케팅/세일즈 타겟팅 전략을 수립해야 할 때
- 고객 세그먼테이션이 필요할 때

## 동작
1. **기존 데이터 분석**:
   - 현재 최고 고객의 공통 특성 식별
   - 높은 LTV, 빠른 활성화, 낮은 이탈률 고객 분석
   - 반대로 이탈한 고객의 공통 특성도 분석
2. **ICP 프레임워크 작성**:
   - 인구통계: 산업, 회사 규모, 지역, 직무
   - 행동 특성: 기술 성숙도, 구매 패턴, 의사결정 프로세스
   - 페인 포인트: 핵심 문제, 현재 해결 방법, 불만족 영역
   - 가치 적합성: 제품이 제공하는 가치와 고객 니즈의 일치도
3. **ICP 스코어링 모델**:
   - 각 특성에 가중치 부여
   - 리드/고객 스코어링 기준 수립
   - Tier 분류 (Tier 1: 완벽 적합, Tier 2: 부분 적합, Tier 3: 탐색)
4. **페르소나 연결**:
   - ICP 내 주요 바이어 페르소나 정의
   - 의사결정자, 영향력자, 사용자 역할 구분
   - 각 페르소나의 동기, 우려, 정보 소비 패턴
5. **검증 및 반복**:
   - ICP 가정 검증 방법 제안
   - 분기별 ICP 재검토 프레임워크
   - 시장 확장 시 ICP 진화 로드맵

## 출력
- ICP 문서 (상세 프로필)
- ICP 스코어링 모델
- 바이어 페르소나 카드
- ICP 검증 계획
- Anti-ICP 정의 (타겟하지 않을 고객)

## 참고
- Dart 풀스택을 사용하는 CoCode 제품의 특성상, 기술 친화적 고객 세그먼트 우선 고려
- Flutter 앱 사용자와 Jaspr Web 사용자의 ICP가 다를 수 있음 (모바일 우선 vs 웹 우선)
- B2B 제품의 경우 Serverpod 기반 엔터프라이즈 기능 (보안, 확장성)이 ICP 선정에 영향
- BMAD의 사용자 스토리와 별개로, GTM 관점의 구매자 프로필에 집중
