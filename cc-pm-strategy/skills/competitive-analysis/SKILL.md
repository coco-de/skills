# Competitive Analysis (경쟁 분석)

## 트리거
- 시장 내 경쟁 상황을 파악해야 할 때
- "경쟁 분석", "competitive analysis", "시장 분석", "차별화" 키워드가 언급될 때
- 제품 포지셔닝 전략을 수립해야 할 때
- 새로운 시장 진입 여부를 판단해야 할 때

## 동작
1. **경쟁사 식별 및 분류**:
   - 직접 경쟁자 (같은 문제, 같은 솔루션)
   - 간접 경쟁자 (같은 문제, 다른 솔루션)
   - 대체재 (다른 방식으로 동일 니즈 충족)
   - 잠재 진입자 (인접 시장 플레이어)
2. **시장 매핑**:
   - 경쟁 포지셔닝 맵 (2x2 매트릭스) 설계
   - 축 선정 (가격/기능, 복잡성/타겟 등)
   - 빈 공간 (white space) 식별
3. **경쟁사 심층 분석**:
   - 기능 비교 매트릭스
   - 가격/패키징 비교
   - 기술 스택 분석
   - 사용자 리뷰 및 평판 분석
   - 성장 전략 및 최근 동향
4. **차별화 전략 수립**:
   - 지속 가능한 경쟁 우위 식별
   - 차별화 유형 분석 (기능, 경험, 가격, 통합, 브랜드)
   - "10x better" 영역 정의
5. **전략적 시사점 도출**:
   - 위협 및 기회 요약
   - 방어 전략 제안
   - 공격 전략 제안

## 출력
- 경쟁사 프로파일 문서
- 경쟁 포지셔닝 맵
- 기능 비교 매트릭스 (Markdown 테이블)
- 차별화 전략 문서
- SWOT 분석

## 참고
- Dart 풀스택의 "하나의 언어로 전체 스택" 접근은 그 자체로 차별화 요소
- Flutter 기반의 크로스 플랫폼 네이티브 성능은 React Native, Ionic 등과 비교 포인트
- Serverpod vs 다른 BaaS/서버 프레임워크 (Firebase, Supabase, Node.js) 비교 관점 포함
- Jaspr Web의 Dart-to-JS 컴파일 접근은 기존 웹 프레임워크와의 차별점
